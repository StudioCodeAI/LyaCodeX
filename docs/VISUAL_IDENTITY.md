# Visual Identity — LyaCodex II

> **O que agrada os olhos, agrada o coração.**

A experiência visual do LyaCodex II não é enfeite. Ela é parte da confiança do usuário.

## Princípio

O LyaCodex II deve parecer:

- sólido;
- moderno;
- leve;
- técnico;
- elegante;
- vivo;
- confiável.

A interface deve fazer o programador sentir que está diante de uma ferramenta séria, não de um experimento improvisado.

## Direção visual

### 1. Clareza acima de excesso

Evitar poluição visual. Cada painel deve ter função clara.

### 2. Identidade viva

A LyaCodex deve respirar visualmente:

- pulsos sutis;
- transições suaves;
- estados de wake;
- streaming fluido;
- feedback de execução.

### 3. Hierarquia forte

O usuário deve entender rapidamente:

```text
recomendado
local
cloud
híbrido
perigoso
seguro
```

### 4. OpenAI GPT Local como destaque

O primeiro motor recomendado deve ter presença especial:

```text
⭐ OpenAI GPT Local
```

Ele deve parecer a opção natural para o primeiro sopro da Lya.

### 5. Terminal em primeiro lugar

O visual nunca pode atrapalhar o terminal.

A UI deve apoiar o fluxo do programador, não competir com ele.

## Frases de identidade

```text
Antes de executar, a Lya precisa respirar um motor.
```

```text
A Lya está onde o código acontece: no terminal.
```

```text
Se você pensa, você executa.
Se você executa, você indexa.
Se você indexa, você evolui.
```

## Regra final

Uma ferramenta poderosa que parece frágil perde confiança.

Uma ferramenta poderosa que parece sólida cria desejo de uso.

> O que agrada os olhos, agrada o coração.
